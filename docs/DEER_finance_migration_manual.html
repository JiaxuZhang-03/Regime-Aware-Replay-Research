01

先做哪一个版本

不要同时复现 detector、DoE、SAC 和金融环境。第一版只验证核心因果链。

推荐：Project MVP

  • 保留当前统一的 DQN 环境、reward 和网络。
  • 直接读取现有三种 regime labeler 给出的 change points。
  • 在 boundary 保存旧 Q snapshot，计算 normalized DoE。
  • 只替换 replay priority，比较 Uniform、PER、Regime Replay、DEER。

第二步：Paper-faithful DEER

  • 使用相邻 reward-sequence windows。
  • 训练二分类器并计算 density-ratio / JS score。
  • 由检测分数在 TD-error 和 DoE 之间动态混合。
  • 比较外部 regime detector 与 DEER 内生 detector。
论文叙事：先问“DoE priority 在金融 non-stationarity 下是否有效”，再问“何种 regime detector 最适合触发 DoE”。两步分开，结果才可解释。

03

先冻结金融环境契约

DEER 只能改变 replay；如果同时改 reward、action 或特征，无法归因。

State

  • 仅使用当时可得的 rolling features。
  • 必须包含当前持仓或目标仓位。
  • normalizer 只在训练历史上拟合。
  • 所有方法使用完全相同的输入。

Action

  • 推荐目标仓位，而非重复 buy/sell 命令。
  • 最简：short / flat / long。
  • 明确仓位上限、换仓时点和执行价格。
  • 禁止不同 replay 方法使用不同 action mask。

Reward

  • 推荐 net log return。
  • 交易成本直接进入 portfolio value。
  • 第一轮不加入 Sharpe、MDD 等复杂项。
  • 记录 raw reward 与 training-scaled reward。
r_t = log(V_{t+1} / V_t),    V_{t+1} includes transaction cost
最低要求：Uniform Replay 必须先能在多个 seed 下学出非随机、非全持有策略。否则 DEER 只是在重新排列噪声。
04

Regime change 如何触发

项目 MVP 和论文忠实版应当作为两个独立实验，不要混成一个黑盒。

A. 现有 labeler 触发

MVP低风险

  1. label 从 (z_{t-1}) 变为 (z_t) 时触发 boundary。
  2. 保存 `q_reference` snapshot。
  3. 为 transition 保存 boundary_id；采样时再动态判断 current / old。
  4. 用 detector 的实际发布日期，禁止使用平滑后的未来标签。
  5. MVP 只要求输入 date, regime_label；label 必须在决策时刻可获得。

B. DEER reward-window detector

Faithful独立消融

两个固定长度但持续滑动的相邻窗口：

reference: [t-2m+1, t-m]   |   test: [t-m+1, t]

分类器区分两组 reward sequences；density-ratio score (S) 超过阈值时，在约 (t-m) 处报告变化。

金融特别风险：agent reward 同时受市场和当前策略影响。若策略刚好开始频繁换仓，reward distribution 会变化，分类器可能把“策略变了”误判为“市场变了”。建议额外比较 market-return 或 volatility sequence detector。
05

如何计算金融 DoE

同一条 transition 是“测试题”；变化前后的两个 Q estimator 分别回答。

1. Detect发现 boundary (T_i)
2. Snapshot冻结变化前 critic 为 (Q_{old})
3. Adapt用新 regime 数据更新当前 critic
4. Probe同一 ((s_k,a_k)) 输入两个 critic
5. Normalize得到可比较的 normalized DoE
DoE_k = | Q_current(s_k, a_k) - Q_reference(s_k, a_k) |

DQN 推荐实现

  • boundary 时复制一张冻结的 Q 网络。
  • 对离散动作直接 gather transition 中实际执行的 (a_k)。
  • `q_reference` 不参与 optimizer，也不做 target update。
  • 每次被采样时 lazy refresh DoE，避免全 buffer 每步重算。

必须写进 limitation

  • 理论 DoE 要求相同 policy，实际 DQN policy 会随 critic 改变。
  • 旧 state 在新 regime 下可能是 OOD。
  • Q 差还包含函数逼近误差和随机优化噪声。
  • 因此第一版应称 empirical Q-discrepancy。

金融尺度修正

MAD(d) = median(|d - median(d)|)
scale_hat = median(d) + 1.4826 × MAD(d) + ε
scale_t = ρ × scale_(t-1) + (1-ρ) × scale_hat
zDoE_k = clip(DoE_k / scale_t, 0, z_max)

这里使用 scale-only robust normalization，不减去 median，因此保留 DoE = 0 → zDoE = 0，并保证 priority 随 DoE 单调变化。常数 1.4826 = 1 / Φ⁻¹(0.75)，作用是让正态分布下的 MAD 具有近似标准差的尺度；它不是需要调节的超参数。

跨 minibatch 保持尺度：不要用当前 minibatch 自己计算 scale。每隔 K_scale 次 gradient update，从 replay buffer 均匀、无放回抽取独立 probe set，计算 scale_hat，再用 EMA 更新全局 scale_DoE。两个 refresh 之间所有 minibatch 共用同一个 scale。默认 K_scale=5，因为按日频每步只更新一次时，500 updates 会晚于整个 S 衰减窗口。
  • 首次初始化：用第一个有效 probe set 直接赋值；若 scale_hat 近似 0，则跳过本次更新。
  • 新 boundary：已有 scale_DoE 不清零；第一次 boundary 尚无尺度时，在 critic 首次更新且 DoE 非零后强制做一次 probe refresh，不必等满 K_scale。
  • 为 |TD| 维护独立的 scale_TD，不要与 DoE 共用尺度。
  • 默认 z_max=5；sigmoid 在约 0–4 最有区分度，超过 5 后接近饱和。
06

Priority 规则

旧经验强调“仍然适用”，新经验在“学习误差”和“环境差异”之间切换。

Pre-change transitions

p_old(k) = 2σ(-λ · zDoE_k) + ε

DoE 小：旧经验在新环境中价值没有明显改变，优先保留。DoE 大：经验可能已过时，降低采样概率。

Post-change transitions

p_new(k) = (1-S)[2σ(zTD_k)-1] + S[2σ(λzDoE_k)-1] + ε

S 高时强调环境差异；S 低时回到 TD-error refinement。原文中的 change score 在触发检测时已经达到 threshold，并非从 0 开始。MVP 用 external label 确认变化，因此在 boundary 直接给较高的 S0，再按新 transition 数量衰减。

age_new = max(n_new - 1, 0)
S(n_new) = S0 × 2^(-age_new / H)
if S < S_floor: S = 0

n_new 是当前 boundary 后收集的不同 transition 数量；同一 transition 被 replay 多次不能重复计数。减去 1 是为了让第一条新 transition 确实使用完整的 S0。每次 external label 改变时重置 n_new=0。默认 S0=0.8、H=5、S_floor=0.05。

默认轨迹：n_new = 1 / 6 / 11 / 16 / 21 时，S = 0.80 / 0.40 / 0.20 / 0.10 / 0.05；之后置 0。

S 的预期轨迹：n_new=1/6/11/16/21 时，S≈0.80/0.40/0.20/0.10/0.05。这模拟原文“检测时已经较高，随后逐渐回到 PER”的行为。
P(k) = p_k^α / Σ_j p_j^α     w_k = (N·P(k))^-β / max_j(w_j)
量作用起始值注意
(α)priority 对 sampling 的影响强度0.6必须包含 α=0 的 Uniform 消融
(β)importance-sampling bias correction0.4 → 1.0随训练进度 anneal
(λ)DoE sigmoid 灵敏度1.0只在 normalized DoE 上使用
(ε)避免 transition 永远不被采样1e-6所有 priority 都必须为正
刚切换时 DoE 可能仍接近 0：新 transition 插入 buffer 时必须暂时使用当前最大 priority，确保它至少被学习一次；之后再用最新 TD/DoE 刷新。由于 S0=0.8 而不是 1，TD 分支也始终保留 20% 权重。

Replay buffer 数据结构

多存几列元数据，才能复现 priority、诊断 regime bias，并支持后续消融。

TransitionRecord:
    state
    action
    reward_raw
    reward_scaled
    next_state
    done
    timestamp
    regime_id
    boundary_id
    td_error
    q_reference_value
    q_current_value
    doe_raw
    doe_normalized
    priority
    last_priority_update_step

Replay manager 还需要保存全局状态：current_regime、current_boundary、n_new、S、scale_DoE、scale_TD、last_scale_refresh_step。

不要永久保存 is_post_change：下一次 regime 切换后它会失效。采样时动态计算 is_current = (transition.boundary_id == current_boundary)。

插入时

新 transition 使用当前最大 priority，避免因尚未计算 TD/DoE 而永远抽不到。

采样时

按 sum-tree 或 segment tree 抽样，返回 index 与 IS weight。

更新后

用最新 TD-error 和 DoE 刷新被采样 transition 的 priority，并记录更新时间。

推荐训练算法

下面是适合当前 DQN 项目的可实现版本，而不是对论文伪代码的逐字照搬。

initialize online_q, target_q, replay_buffer
q_probe = ema_copy(online_q)  # Use target_q instead only when it updates fast enough.
q_reference = frozen_copy(q_probe)
current_regime = None
current_boundary = 0
n_new = 0
S = 0
scale_doe = None
scale_td = None
force_scale_refresh = False
gradient_update_step = 0

for each market step t:
    # External label must be known at decision time t; no future smoothing.
    regime = external_labels[t]
    changed = current_regime is not None and regime != current_regime

    if changed:
        current_boundary += 1
        q_reference = frozen_copy(q_probe)
        n_new = 0
        S = S0
        force_scale_refresh = True
    current_regime = regime

    state = build_state_without_future_information(t)
    action = epsilon_greedy_target_position(online_q, state)
    next_state, reward, done = env.step(action)

    replay_buffer.add(
        state, action, reward, next_state, done,
        timestamp=t, regime_id=regime,
        boundary_id=current_boundary,
        priority=replay_buffer.max_priority
    )

    if current_boundary > 0:
        n_new += 1  # Count unique newly collected transitions, not replays.
        age_new = max(n_new - 1, 0)  # The first new transition uses full S0.
        S = S0 * 2 ** (-age_new / H)
        if S < S_floor:
            S = 0

    if replay_buffer.ready():
        batch, indices, is_weights = replay_buffer.sample(alpha)

        q_pred = online_q(batch.state).gather(batch.action)
        q_target = double_dqn_target(batch, online_q, target_q)
        td = q_target - q_pred

        with no_grad:
            q_old = q_reference(batch.state).gather(batch.action)
            q_new = q_probe(batch.state).gather(batch.action)
            doe = abs(q_new - q_old)
            if scale_doe is None or scale_td is None:
                priority = abs(td) + eps  # PER fallback during initialization.
            else:
                z_td = clip(abs(td) / scale_td, 0, z_max)
                z_doe = clip(doe / scale_doe, 0, z_max)
                post = batch.boundary_id == current_boundary
                priority = where(
                    post,
                    (1-S) * (2*sigmoid(z_td)-1)
                        + S * (2*sigmoid(lambda_doe*z_doe)-1) + eps,
                    2*sigmoid(-lambda_doe*z_doe) + eps,
                )

        loss = mean(is_weights * huber(td))
        optimize online_q with gradient clipping
        ema_update(q_probe, online_q, tau_probe)
        gradient_update_step += 1
        replay_buffer.update_priorities(indices, priority)
        periodically_update(target_q)

    regular_refresh = gradient_update_step > 0 and gradient_update_step % K_scale == 0
    if replay_buffer.ready() and (regular_refresh or force_scale_refresh):
        probe_size = min(M_probe, len(replay_buffer))
        probe = replay_buffer.uniform_sample_without_replacement(probe_size)
        doe_probe, td_probe = evaluate_without_grad(probe)
        scale_td = rolling_robust_scale(abs(td_probe), scale_td, rho)
        if current_boundary > 0 and any(doe_probe > scale_floor):
            scale_doe = rolling_robust_scale(doe_probe, scale_doe, rho)
            force_scale_refresh = False

rolling_robust_scale(values, previous, rho):
    median_value = median(values)
    mad = median(abs(values - median_value))
    candidate = median_value + 1.4826 * mad + eps_scale

    if candidate <= scale_floor:
        return previous
    if previous is None:
        return candidate
    return rho * previous + (1-rho) * candidate
为什么 DoE 用 target / EMA critic：online Q 每个 gradient step 都抖动，直接比较会把优化噪声当环境变化。若 target network 在 H 条新 transition 内至少更新一次，可直接使用；否则推荐单独维护每次 critic update 都更新的 EMA probe critic（默认 τ_probe=0.01），并在 boundary 冻结它作为 q_reference。
实现顺序检查：必须先读取 external label 并处理 boundary，再把当天 transition 入库；否则第一条新-regime transition 会被错误标成旧数据。初始阶段若 scale_DoE/scale_TD 尚未建立，先使用 PER priority，直到第一个有效 probe scale 可用。

两周实现与验收

按依赖关系推进：先建立可信 baseline，再接入 DoE，最后扩展 detector。

阶段 1：2–3 天

  • 冻结 environment contract。
  • 稳定 Uniform / PER。
  • 完成 diagnostics logging。
  • 5 seeds 小规模 sanity run。

阶段 2：3–5 天

  • 实现 external-label boundary 与 Q snapshot。
  • 实现 rolling DoE/TD scale。
  • 实现 S transition-count schedule。
  • 接入 priority 并检查 sampling。

阶段 3：4–7 天

  • 完整主实验和消融。
  • S 与 scale 稳健性。
  • transaction cost 与 regime 稳健性。
  • 生成论文图表和结果表。

提交代码前的最小测试

  • external label 不变时不创建新 boundary；变化时只 snapshot 一次并重置 n_new=0, S=S0。
  • 第一条新 transition 后 S=0.8；第 6 / 11 / 16 / 21 条后约为 0.4 / 0.2 / 0.1 / 0.05；replay 抽样不改变 n_new。
  • q_reference 在同一 boundary 内参数完全不变，q_probe/q_current 每次 critic update 按 EMA 改变。
  • 同一 transition 在不同 minibatch 中、两次 scale refresh 之间使用相同的 scale_DoE/scale_TD。
  • probe set 均匀无放回；全零 probe 不覆盖上次有效 scale；buffer 小于 2048 时不会采样越界。
  • p_old 随 zDoE 单调下降；S=0 时 p_new 等价于 normalized PER 分支；所有 priority 有限且大于 0。
  • regime label、state 和 reward 在决策时均不含未来数据，训练/验证/测试按时间切分。
MVP 验收标准：相同 DQN、数据、reward 和训练预算下，DEER 在至少 5 个 seed 中提高 median shift-recovery speed 或降低 worst-regime loss；同时没有通过极端 turnover 获得表面收益。
停止条件：若 Uniform DQN 仍无法稳定超过 random/cash 或形成有意义的 action distribution，暂停 DEER 大实验，先修环境。Replay 方法不能替代可学习的交易信号。
