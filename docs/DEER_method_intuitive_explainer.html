DEER Method / 直觉讲解

Sample Efficient Experience Replay in Non-stationary Environments
今晚只回答一个问题：环境变了以后，replay buffer 应该复习哪些经验？

PER 的信号

TD-error

agent 对这条 transition 预测错了多少。

DEER 新增的信号

DoE

环境变化让这条 state-action 的价值改变了多少。

最终目标

变化期先适应新环境，稳定后再选择性复用旧经验。

一句话：DEER 不是简单地“忘记旧数据”，而是区分哪些旧经验已经过时、哪些旧经验跨环境仍然有用。
讲述提示：先不要讲公式。用“地面从干燥变湿滑，过去走路经验有些失效、有些仍适用”的例子开始。

1 / 为什么 PER 在环境变化后可能出错

Stationary 情况

高 TD-error → agent 还没有学好 → 多复习通常有价值

这正是 Prioritized Experience Replay 的核心逻辑。

Non-stationary 情况

高 TD-error → 可能是 agent 没学好 → 也可能是环境已经变了

PER 无法判断这两个原因，可能不断抽取已经不适用于当前环境的旧 transition。

旧环境同一动作产生正收益
→
Regime Shiftdynamics / reward 改变
→
PER旧经验 TD-error 变大，反而被频繁抽取
DEER 的出发点：高 TD-error 不等于高 current-environment relevance。
讲述提示：联系我们的结果：PER mismatch 为 0.5785，比 Uniform 的 0.5674 还高，与这一直觉一致，但还不能证明因果。

2 / 第一步：先检测环境是否变化

DEER 不直接观察 friction 或 damping，而是比较相邻时间窗口的 reward sequence distribution。

讲述提示：这里的 S 既是 change detector，也是后面 TD-error 和 DoE 之间的动态 gate。

3 / 第二步：定义 Environment Discrepancy

DEER 想估计：同一个 state-action 的价值变化，有多少来自环境本身变化。

DoE(s, a) = Q_new(s, a) - Q_old(s, a)

低 |DoE|

Q_old(s,a) = 10

Q_new(s,a) = 9.8

环境虽然变了，但这条经验对应的价值关系基本没变，旧 transition 仍可能有用。

高 |DoE|

Q_old(s,a) = 10

Q_new(s,a) = -3

同一动作在新环境中的价值完全改变，旧 transition 很可能已经过时。

信号想回答的问题高值的直觉
TD-erroragent 对当前 Bellman target 预测错多少？agent 还需要继续学习这条经验
DoE环境变化让 state-action value 改变多少？这条经验对 dynamics shift 很敏感
讲述提示：强调 DoE 是概念上的分离，不是完美识别。Q_old 和 Q_new 都可能估计错误。

4 / 第三步：旧经验和新经验分开处理

Change 之前的旧 transition

priority_old = 2σ(-|DoE|)
  • 低 DoE：旧经验跨环境仍然成立，优先保留。
  • 高 DoE：旧经验受环境变化影响大，降低 priority。
  • 不是全部遗忘，而是 selective reuse。

Change 之后的新 transition

priority_new = (1-S) × normalized TD + S × normalized |DoE|
  • S 高：环境刚变化，优先适应新 dynamics。
  • S 低：环境稳定，重新偏重 TD-error。
  • 从 environment adaptation 自动切回 policy refinement。
DEER 最关键的设计不是 DoE 本身，而是：旧数据和新数据使用不同 priority，并且新数据的 priority 会随变化阶段动态调整。
讲述提示：对比我们的固定 quota：我们目前按 same-regime / high-TD / recent / random 固定混合，DEER 的权重会随变化分数 S 自动改变。

5 / DEER 在时间轴上如何工作

Stage A：变化之前

环境稳定，正常利用 replay。

旧经验大多仍符合当前 dynamics。

normal learning
Stage B：刚发生变化

reward-window discrepancy 上升，检测到 change point。

重点采样能反映新 dynamics 的 post-change transition。

rapid adaptation
Stage C：逐渐稳定

变化分数 S 下降，priority 逐渐回到 TD-error。

选择性复用低 DoE 的旧经验，维持稳定和长期记忆。

selective reuse
先学新规则增加 post-change priority
→
确认哪些旧知识失效利用 DoE 过滤旧 transition
→
恢复正常学习重新强调 TD-error 与长期记忆
白话类比：换到新公司后，第一周先集中学新流程；熟悉以后，再把上一份工作里仍通用的技能拿回来，而不是把旧知识全部删除。

6 / 完整训练循环

StepDEER 做什么直觉
1agent 与环境交互，将 transition 放入 replay buffer正常收集经验
2维护 reference 和 test reward windows观察最近环境是否和过去不同
3训练 binary MLP，计算 JS discrepancy score S判断是否发生 shift
4超过 threshold 后标记 change point，并保留旧 Q-function 信息区分 change 前后的 transition
5比较 Q_new 与 Q_old，计算 DoE判断经验受 dynamics 改变的程度
6旧 transition 按低 DoE 优先保留跨环境可迁移经验
7新 transition 按 S 动态混合 DoE 和 TD-error在 adaptation 与 policy learning 间切换
8按 priority 抽样并使用 importance weight 更新 SAC纠正非均匀采样偏差
9持续更新 Q、TD-error、DoE 和 priority让采样策略随训练阶段变化
P(i) = priority_i^α / Σ priority^α w_i ∝ (N × P(i))^(-β)

论文实验使用 α = 0.6、β = 0.4。这里和 PER 一样，通过 importance sampling weight 减少 prioritized sampling 带来的估计偏差。

7 / Method 的关键假设与风险

DEER 默认依赖什么

  • reward distribution 的变化能够提示 environment shift。
  • change point 能通过固定窗口和 threshold 被检测。
  • Q_old 与 Q_new 足够可靠，可以用来计算 DoE。
  • 新旧环境存在可比较、可复用的 state-action structure。

放到金融里会遇到什么

  • reward 变化可能来自市场、policy、仓位、成本或随机噪声。
  • regime 往往渐进、反复，没有干净 change point。
  • 废 policy 会产生不可靠的 Q，DoE 可能只是两个错误估计的差。
  • 金融相关性需要通过 out-of-sample 风险和收益证明。
所以 DEER 提供的是一个很强的 replay 设计思想，但不是可以不加验证地直接搬到金融市场的答案。
讲述提示：这页是连接到我们项目的桥梁：我们的 explicit regime label 更可解释，但 DEER 的动态 priority 更有方法深度。

8 / 如何把 DEER 移植到金融环境

Step金融环境中的实现必须注意
1. 定义金融 MDP 固定 state、action、reward 和 transaction cost。先沿用当前 DQN 环境，确保 Uniform/PER 能稳定运行。 先检查 portfolio accounting、cash、drawdown、reward scale；废 policy 会让后续 DoE 失去意义。
2. 构造 change detector 使用相邻 reward windows 训练 binary MLP，计算 JS score S；同时保留 rule/HMM/CUSUM label 作为外部验证。 金融 reward 噪声大，变化可能来自仓位、policy 和成本，不一定来自 market dynamics。所有窗口只能使用当时及过去数据。
3. 保存 Q snapshot 检测到 change point 时冻结一份 Q_old，当前网络继续学习为 Q_new，在 buffer transition 上计算 |Q_new-Q_old|。 Q 网络必须在 change 前已有基本学习能力；否则 DoE 只是两个不可靠 Q 估计的差。还要固定 normalization 和 action definition。
4. 接入 DEER priority change 前 transition 按低 DoE 优先；change 后 transition 用 S 动态混合 DoE 与 TD-error，再使用 PER 式 sampling 和 importance weights。 保留随机样本和 priority floor，避免只抽少数 transition；记录 pre/post-change sampling share 和 effective sample diversity。
5. 做公平实验 同一 DQN、数据、seed、buffer 和交易成本下比较 Uniform、PER、当前 Regime-aware、DEER-like，以及可选 Hybrid。 严格 chronological train/validation/test；不要用未来 regime label；除收益外报告 recovery、drawdown、turnover、mismatch 和 detector false alarms。

建议分两级移植

Level 1：DEER-like MVP

先用已有 regime switch 作为 change point，重点验证 Q snapshot、DoE priority 和动态 sampling 是否能运行。

目的：分离“priority 设计问题”和“change detector 问题”。

Level 2：Faithful DEER

再加入 reward-window MLP detector，让模型自己发现 shift，并与 rule/HMM/CUSUM 检测结果对照。

目的：检验通用 DEER detector 在高噪声金融环境中是否可靠。

最关键的实验原则：先证明 DQN 与 Q-value 基本可信，再测试 DoE；先用已知 change point 验证 sampling，再加入 learned detector。不要同时更换环境、detector、priority 和 RL algorithm。

9 / 两分钟总结话术

DEER 认为，在非平稳环境里，TD-error 大不一定表示经验有价值，也可能只是环境已经变化。它先比较相邻 reward windows，用二分类器和 JS divergence 检测 dynamics shift；再通过新旧 Q-function 的差定义 DoE，估计每条 transition 受环境变化影响的程度。对于 change 前的旧经验，低 DoE 的被保留；对于 change 后的新经验，环境刚变化时重点看 DoE，环境稳定后逐渐重新看 TD-error。这样 agent 先快速适应新环境，再选择性复用仍然有效的旧经验。
对我们来说，DEER 已经占据了 general non-stationary replay 的位置。我们的区别是用明确的 financial market regime 作为 relevance signal，更简单、更可解释。但现在只证明 mismatch 下降还不够，因为按 same-regime 抽样天然会降低 mismatch。下一步必须证明它改善 post-switch recovery、drawdown、turnover 或 out-of-sample stability，并考虑把固定采样比例改成类似 DEER 的动态机制。

讨论时建议问伙伴的三个问题

  • 我们是否把 DEER-like method 加成必需 baseline？
  • 固定 same-regime ratio 是否应该改成 change-aware dynamic ratio？
  • 在 Q-function 尚不可靠时，我们先修 DQN，还是先只做 synthetic mechanism validation？
