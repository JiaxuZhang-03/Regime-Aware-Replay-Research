DQN Replay 实验运行与评估说明

适用项目：Regime-Aware-Replay-Research ｜ 内容：不同 regime label 的运行命令、实验评估标准、模型参数说明

0. 实验目标

本实验的核心不是证明某个交易策略已经足够盈利，而是验证在真实金融非平稳环境中，标准 replay buffer 是否会在当前 market regime 下采样到大量来自其他 regime 的历史 transition，以及 TD-error 优先采样是否会加剧或缓解这种问题。

一句话：比较 DQN-online、DQN + Uniform Replay、DQN + PER、DQN + Regime-aware Replay 在不同 regime label 方法下的表现与 replay mismatch 行为。

2.4 一次性运行三种 label 方法

PowerShell 中可以直接运行：

    python scripts\run_dqn_replay.py --label-method $label --replays online,uniform,per,regime --seeds 0,1,2

如果只想快速补跑某个方法，可以修改 --replays，例如只跑 regime-aware replay：
python scripts\run_dqn_replay.py --label-method rule_based --replays regime --seeds 0,1,2

3. 四种方法的含义

方法名 完整含义 实验作用
online DQN-only / no replay 不用 replay buffer，只用最新 transition 更新。作为没有历史经验回放的基础参照组。
uniform DQN + Uniform Replay 从 replay buffer 中等概率随机抽样，用于验证普通 replay 是否会混入大量 mismatched-regime transitions。
per DQN + Prioritized Experience Replay 按 TD-error 大小优先抽样，用于验证 TD-error priority 在金融非平稳环境下是否会偏向旧 regime 的高误差样本。
regime DQN + Regime-aware Replay 采样时显式考虑当前 regime，按 same-regime、high-TD、recent、random 的比例混合抽样，用于降低 mismatch rate。

4. 实验评估标准

4.1 交易表现指标

交易表现指标用于判断 agent 作为交易策略的基本表现，但不是本实验唯一重点。

指标 方向 含义 输出列
Final Portfolio Value 越高越好 最终组合净值，初始值为 1。 final_portfolio_value
Max Drawdown 越低越好 最大回撤，衡量风险控制能力。 max_drawdown
Mean Turnover 适度较低更好 平均换手率。过高说明交易过于频繁，交易成本压力大。 mean_turnover
Mean Reward 越高越好 平均单步 reward，即组合 log return 扣除交易成本后的均值。 mean_reward

4.2 Replay 机制诊断指标

这部分才是本项目当前阶段的核心，用于判断 replay 是否存在 regime contamination。

指标 方向 含义 输出列
Mismatch Rate 越低越好 采样 batch 中 sampled_regime != current_regime 的比例。 mean_mismatch_rate
Sample Age 需结合 regime 判断 被采样 transition 距离当前时间的步数。不是越小越好，但过旧样本可能更容易过时。 mean_sample_age, median_sample_age
TD-error by Regime 用于解释采样行为 不同 regime 样本的 TD-error。如果旧 regime 的 TD-error 高，PER 可能更容易抽到旧样本。 td_error_regime_*
Priority by Regime PER / regime 专用 观察 priority 是否集中在某些 regime 上。 priority_regime_*, mean_priority
Source Composition regime 专用 Regime-aware replay 中 same-regime、high-TD、recent、random 样本各占多少。 source_same_regime_count, source_high_td_count, source_recent_count, source_random_count
判断逻辑：如果某个方法收益更高，同时 mismatch rate 更低、drawdown 更小、turnover 不明显升高，说明它更适合 regime-shifting market。若收益提升但 mismatch rate 和旧 regime priority 也升高，则说明它可能没有真正解决 replay contamination。

5. 结果解读模板

现象 解释
uniform mismatch rate 高 说明标准随机 replay 会把不同 market regimes 的历史 transition 混在一起训练。
per TD-error 高且 mismatch rate 高于 uniform 说明 TD-error priority 可能偏向旧 regime 的高误差样本，支持“TD-error 在非平稳金融环境中不一定可靠”的论点。
regime mismatch rate 明显下降 说明 regime-aware sampling 有效降低了 replay contamination。
regime final value 提升、max drawdown 下降 说明降低 mismatch 不只是日志现象，也可能改善实际交易适应性。
online 表现较弱但 mismatch 为 N/A 这是合理的。online 没有 replay buffer，因此没有 mismatch rate，但它的 sample efficiency 通常较差。

6. 模型与实验参数说明

6.1 数据与标签参数

参数 默认值 说明
--market-csv data/market_indices_20080601_20260531/market_regime_features_wide.csv 真实市场数据路径。
--labels-csv outputs/regime_labels/all_regime_labels.csv 已生成的 regime label 文件。
--label-method rule_based 选择使用哪种 regime label，可选 rule_based、hmm、recap_cusum。
--tradable-symbols DIA,SPY,QQQ DQN 可交易资产。
--primary-symbol SPY 构造市场特征时的主参考资产。

6.2 交易环境参数

项目 默认设定 说明
动作空间 cash / DIA / SPY / QQQ / equal_weight DQN 使用离散动作，而不是连续权重分配。
Reward log(V_t / V_{t-1}) - transaction_cost 组合 log return 扣除交易成本。
--transaction-cost-bps 10.0 交易成本，10 bps = 0.1%。
State features ret_short, ret_long, vol, trend, vix, turbulence + previous action 实际使用哪些列取决于数据中存在的特征。

6.3 DQN 参数

参数 默认值 说明
--gamma 0.99 折扣因子，越接近 1 越重视长期收益。
--lr 1e-3 学习率。
--hidden-dim 128 Q-network 两层 MLP 的隐藏层维度。
--target-update-freq 500 每 500 次参数更新同步一次 target network。
--epsilon-start 1.0 初始探索概率。
--epsilon-end 0.05 最低探索概率。
--epsilon-decay-steps 8000 epsilon 从 1.0 衰减到 0.05 的步数。

6.4 Replay 参数

参数 默认值 适用方法 说明
--buffer-size 50000 uniform, per, regime Replay buffer 最大容量。
--batch-size 64 uniform, per, regime 每次训练从 buffer 中抽取的 transition 数量。
--warmup-steps 256 uniform, per, regime buffer 至少积累多少 transition 后开始训练。

6.5 PER 参数

参数 默认值 说明
--per-alpha 0.6 priority 强度。越大越偏向高 TD-error 样本。
--per-beta-start 0.4 importance-sampling correction 的初始强度。
--per-beta-end 1.0 importance-sampling correction 的最终强度。
--per-eps 1e-6 防止 priority 为 0。

6.6 Regime-aware Replay 参数

参数 默认值 含义
--regime-same-ratio 0.50 每个 batch 中优先抽取当前 regime 样本的比例。
--regime-high-td-ratio 0.25 每个 batch 中按 TD-error priority 抽样的比例。
--regime-recent-ratio 0.15 每个 batch 中抽取近期样本的比例。
--regime-random-ratio 0.10 每个 batch 中随机样本比例，用于保持多样性。
--regime-recent-window 252 recent samples 的窗口长度，252 约等于一年交易日。

7. 推荐报告表格

最终汇报时，可以对每个 label method 单独给出一张表：

Label Method Replay Method Final Value ↑ Max Drawdown ↓ Turnover ↓ Mismatch Rate ↓ TD-error Priority
rule_based online N/A N/A
rule_based uniform N/A
rule_based per
rule_based regime
最终结论应围绕：不同 label 方法下，regime-aware replay 是否稳定降低 mismatch rate；PER 是否因 TD-error 机制抽到更多 mismatched-regime 样本；这些 replay 行为是否和 drawdown、turnover、final portfolio value 有关。

8. 文献与项目依据

  • ReCAP 将金融投资问题明确放在 regime-shifting continual portfolio management 场景下，使用 adaptive regime detection、policy vector library 和 regime gate 处理市场非平稳性。
  • DEER 指出在非平稳环境中，传统基于 TD-error 的 replay priority 可能会高估旧环境样本的重要性，因为 TD-error 同时受到策略更新和环境变化影响。
  • LILAC 将非平稳环境理解为由隐藏上下文或动态参数驱动的一系列相关 MDP，为“market regime 可以被视为 hidden context”提供概念基础。
  • 本项目当前实现并不复现 ReCAP / DEER / LILAC 的完整模型，而是聚焦在更小的问题：真实金融 regime 下，DQN 的 replay sampling 是否存在 contamination，以及简单 regime-aware sampler 是否能降低 mismatch。
