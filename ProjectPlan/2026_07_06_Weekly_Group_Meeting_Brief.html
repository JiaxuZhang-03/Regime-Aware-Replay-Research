合成市场 Synthetic Market SAC / Replay DEER-style 机制 下周任务

1. 当前项目位置 / Current Position

目前的目标还不是直接证明 trading strategy 能稳定盈利，而是先建立一个可控的 benchmark，把三个问题拆开判断：

Base RL 是否可用

SAC / DQN 能不能在简单 market signal 下学出非退化的 portfolio policy？

Replay 机制是否生效

Regime shift 后，replay method 是否真的改变了被抽样的 transitions？

Evaluation 流程是否清楚

即使 final portfolio return 很 noisy，我们能不能测量 replay 本身的作用？

核心定位：Synthetic market 是 diagnostic environment。它的作用是先检查 regime-aware replay 的行为是否合理，再进入更噪声的真实市场 backtest。

2. Synthetic Market 设计

我们现在已经搭好了一个 progressive synthetic market generator。每一层 level 都增加一种 non-stationarity，这样可以更清楚地判断 replay-aware methods 在什么情况下开始有意义。

当前环境使用 synthetic stocks SYN0 到 SYN4。每日收益由 regime-dependent Gaussian distribution 生成：

r_t | z_t = k ~ N(mu_k, Sigma_k)

  • z_t: hidden market regime。
  • mu_k: 每个 regime 下的 expected return。
  • Sigma_k: 每个 regime 下的 volatility 和 correlation structure。
  • SAC 不直接看到 true regime；replay labels 可以使用 oracle labels，也可以使用 estimated labels。

3. 合成市场难度层级 / Progressive Levels

Level 0

Stationary Market：无 regime shift

所有 returns 都来自一个固定分布。

  • 目的：sanity check。
  • 预期：regime-aware replay 和 DEER 不应该凭空产生优势。
  • 发现：修复 DEER fallback bug 后，在没有 boundary 时，DEER 会正确退化成 PER-like behavior。
Level 1

Two-Regime Mean Shift：只改变均值

两个 regimes：bull 和 bear。只有 expected returns 改变。

  • Bull：大多数股票 expected returns 为正。
  • Bear：大多数股票 expected returns 为负。
  • 目的：测试旧 regime 的 return-direction experience 是否会拖慢 adaptation。
  • 发现：regime replay 明显降低 mismatch；DEER 在短跑 smoke test 中 final value 更好，但还不能当正式结论。
Level 2

Mean + Volatility Shift：均值和波动率一起变

不同 regimes 下 expected return 和 volatility 都改变。

  • Bull：positive mean，lower volatility。
  • Bear：negative mean，higher volatility。
  • 目的：测试 agent 是否能同时适应 return 和 risk 的变化。
  • 发现：regime replay 仍然降低 mismatch；但 portfolio-level improvement 不稳定，说明 risk structure 改变后 base RL 更重要。
Level 3

Mean + Volatility + Correlation Shift：加入相关性变化

Expected return、volatility 和 cross-stock correlation 都随 regime 改变。

  • Bull：lower volatility and lower correlation。
  • Bear：higher volatility and higher correlation。
  • 目的：让 benchmark 更接近 portfolio problem，因为 diversification benefit 会随 regime 改变。
  • 发现：smoke test 中 equal-weight buy-and-hold 强于 SAC，所以 base RL performance 是当前瓶颈。
Level 4

Rare Crisis Regime：加入罕见危机状态

加入低概率 crisis state，特征是 severe negative returns、high volatility、high correlation。

  • Regimes：bull、bear、crisis。
  • 目的：测试 rare but important regime 下的 adaptation 和 drawdown behavior。
  • 发现：SAC 仍然弱于 buy-and-hold；replay diagnostics 仍然有用，但 return comparison 还不可靠。
Level 5

Hidden / Estimated Regime：用估计标签替代真实标签

Underlying market 仍然是 Level 4，但 replay labels 可以来自 estimated regime methods。

  • Oracle labels：true synthetic regime。
  • Estimated labels：HMM 和 ReCAP-CUSUM。
  • 目的：接近真实金融数据，因为真实 regime 在现实里不可观测。
  • 发现：normal-length test 中 DEER 略好于 Uniform/PER，但所有 SAC variants 都明显弱于 buy-and-hold。这说明在做强 return claim 前，base RL 必须先提升。

4. 当前发现 / Current Findings

发现 解释 下一步含义
Regime-aware replay 稳定降低 mismatch。 Sampler 至少在机制上确实改变了抽样分布。 Mismatch reduction 可以作为 mechanism metric。
Uniform 和 PER 的 portfolio results 经常很接近。 PER 是 active 的，但 TD-error priority 目前没有明显改变 portfolio-level outcome。 不能只看 final return，需要 replay diagnostics。
DEER-style replay 目前还比较简单。 当前 DoE、scale、boundary handling 可能还不够 sharp。 应该先在 synthetic market 上调整 replay design。
Base SAC 在困难 levels 上弱于 equal-weight buy-and-hold。 Base model 仍然是主要瓶颈。 Performance tuning 需要和 replay design 并行推进。
主要结论：Synthetic market setup 已经有用，但当前结果主要应解释为 mechanism diagnostics，而不是最终 trading-performance evidence。

5. 下周任务分配 / Task Allocation

下周重点是把 base RL 问题和 replay mechanism 问题拆开，不要所有东西一起调。

Lyra：Replay Design

目标：让 replay mechanism 更合理，并且和 PER 有更清楚的区别。

  • 思考 post-boundary sampling 怎么从 new-regime exploration 过渡到 stable replay。
  • 比较 DoE scale 方案：fixed scale、rolling MAD scale、rank-based DoE。
  • 加入 diversity control，避免 replay 被少数 high-priority noisy samples 垄断。
  • 检查 high-DoE / high-priority samples 是否真的靠近 regime boundaries 或信息更强的区间。
  • 优先用 synthetic Level 4/5 作为 replay behavior benchmark。

简单交付：一张表，对比几个 DEER-style settings 下的 replay diagnostics。

Jiaxu：Base RL Performance

目标：先让 base policy 不那么容易学废，再看 replay comparison。

  • 先固定 Uniform replay，再调 reward scale、transaction cost、action behavior 和 training stability。
  • 检查 policy 是否塌缩到 cash、equal-weight、single asset 或 excessive turnover。
  • 记录 reward scale、Q-value scale、actor/critic losses 和 action distribution。
  • 先用简单的 synthetic Level 1/2 验证，再判断 Level 4/5。

简单交付：一份 stability report，说明 SAC 是否能学出 non-degenerate behavior。

我的任务：Evaluation Pipeline

目标：把 replay effect 的评估流程标准化。

  • 建立每次 experiment run 都能复用的 report format。
  • 把 portfolio metrics 和 replay-mechanism metrics 放在同一份 output 里。
  • 区分哪些实验是 valid，哪些被 weak base RL 卡住，哪些只是 diagnostic。
  • 准备 portfolio value、regime mismatch、sample age、priority、DoE、post-boundary recovery 的图。

简单交付：一个可复用的 experiment summary template。

6. 本周推进原则 / This Week's Decision

不要同时优化所有部分。最清楚的推进方式是一次固定一层，再检查另一层。

层次 本周要回答的问题 不要过度解释
Base RL Policy 能否在简单 synthetic market 中避免 degenerate behavior？ SAC 弱不等于 replay 没用。
Replay DEER-style replay 能否在 regime shift 后产生可解释的 sampling behavior？ Replay diagnostics 更好不自动等于 return 更高。
Evaluation 我们能否稳定测量 adaptation、mismatch 和 replay concentration？ Final value 太 noisy，不能作为唯一指标。
本周判断标准：一个有用的实验，应该能告诉我们失败来自 base RL、replay design，还是 evaluation metric。
